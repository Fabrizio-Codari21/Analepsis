Para crear un caso nuevo se necesita:

1) Traer "--- BASE GAME ---" desde otra escena para que el juego funcione; sumarle a eso "--- MAP ---" donde iria el diseño del nivel y "--- ELEMENTS ---" donde irian los personajes, pistas y demas.

2) En la carpeta "_Game/SO" donde se crean Scriptable Objects (Right Click > Create > Game > [Name]): 
- Crear un CaseResolution (el cual va a contener todas las soluciones posibles al caso en cuestión, cada una con su nombre, descripción, y pistas asociadas a cada rol del whodunnit).

3) Crear personajes:
- Crear una instancia del prefab de BaseCharacter en escena con el mesh deseado.
- Crear un SO de Npc (el cual contiene nombre, rol y aspectos de la estetica y personalidad del npc). 
- Crear un SO de Dialogue (el cual contiene el graph con la conversacion y la info que guarda cada nodo).
- Asignar ambos SO al script NPC del character en escena.
- Meterse al LookAt dentro del Rig y asignar el LookAt del player a la lista de SourceObjects, de modo tal que pueda girar la cabeza hacia el player.

4) Crear pistas: 
- Crear una instancia del prefab de BaseItem en escena con el mesh deseado.
- Crear un SO de Item (el cual contiene nombre, objeto e informacion obtenida en cada punto del analisis). 
- Crear una instancia Prefab_UI del objeto para mostrar durante la inspección, la cual debe tener dentro los POI con el ID correspondiente para poder ser inspeccionados.
- Crear una instancia del maniqui posado con el cual hacer el flashback.
- Crear un TransformKeyInstance dentro de un objeto vacio ya posicionado en la escena (con su propio SO de TransformKey) y meterle el maniqui del flashback adentro.
- Asignar tanto el prefab de UI como el del flashback al SO, y asignar este SO al script ItemReference del item en escena.

5) Una vez creadas todas las pistas y escritos todos los dialogos, asignarlas segun correspondan a la lista de cada solucion del CaseResolution.

6) Listo.
